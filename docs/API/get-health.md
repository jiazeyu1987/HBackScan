# GET `/health`

- 入口：`code/hospital_scanner/main.py:108`
- 作用：返回固定的健康状态 `{status: "healthy"}`。

## 执行流程
1. 路由命中后直接返回常量字典，不检查任何依赖。

## 设计原因
- 作为探活接口供负载均衡/监控使用。

## 优点
- 零依赖，延迟极低。

## 缺点与风险
- 未检查数据库/LLM/任务系统，依赖挂掉时仍会报 healthy，易误导监控。
- 缺少版本、响应时间、组件健康等可观测数据。

## 改进建议
- 增加组件级探测（数据库连接、LLM API、磁盘权限等），失败时返回 500。
- 返回结构化的组件状态，便于监控与告警。
