# GET `/health`

- 入口：`code/hospital_scanner/main.py:108`
- 作用：返回健康状态，当前实现始终返回 `"healthy"`。

## 执行流程
1. 路由命中后直接返回固定字典，不调用数据库、LLM 或任务系统。

## 设计原因
- 作为健康探针供探活/负载均衡使用。

## 优点
- 无依赖，响应快，几乎不会抛异常。

## 缺点与风险
- 未检查数据库、LLM、任务队列等关键依赖，可能在依赖失效时仍报“healthy”，误导监控。
- 缺少响应时间、版本、依赖状态等可观测数据。

## 改进建议
- 增加数据库连接探测、磁盘/权限检查、LLM API 联通性等子检查。
- 将检查结果结构化返回（`status`/`components`），并设置失败时的 HTTP 500。
